# house-prices - predicting house prices
## კონკურსის მიმოხილვა
Kaglle house-prices challenge-ის მიზანია სახლის ფასის პროგნოზირება მრავალ ცვლადზე დაყრდნობით: მდებარეობა, ოთახების რაოდენობა, ფართობი, გარაჟი, გაყიდვის ტიპი და ა.შ.

## Feature Engineering

**1. ახალი ცვლადების შექმნა** 

* შევქმენი ცვლადი HouseAge = YrSold - YearBuilt, რადგან ვფიქრობ ფასზე იმას უფრო აქვს მნიშვნელობა, თუ სახლი რა ასაკისაა, ვიდრე ცალ-ცალკე აღებულ სახლის აგების ან გაყიდვის თარიღს. დავდროპე YrSold და YearBuilt
* ანალოგიურად შევქმენი ცვლადი YearsSinceRemod = YrSold - YearRemodAdd, რაც აღნიშნავს რემონტის ასაკს. დავდროპე YearRemodAdd
* შევქმენი ცვლადი Has2ndFlr, რომელიც გვიჩვენებს აქვს თუ არა სახლს მეორე სართული, რადგან მთლიანი ფართობი და ცალ-ცალკე სართულების ფართობები უკვე გვაქვს და გამოვიყენე სვეტი 2ndFlrSF, თუ მისი მნიშვნელობა 0-ზე მეტია, მაშინ ჰქონია მეორე სართული. თვითონ 2ndFlrSF დავდროპე.

**2. კატეგორიული ცვლადების რიცხვითებში გადაყვანა**

კატეგორიული ცვლადები განსხვავებულია და სხვადასხვანაირად უნდა მოვექცეთ

* ExterQual, ExterCond, BsmtQual, HeatingQC, KitchenQual, FireplaceQu

  ამ ცვლადების მნიშვნელობები ერთმანეთთან შედარებადია, მაგალითად Ex(excellent) > Gd(good). ანუ შეგვიძლია მათი ერთმანეთის მომდევნო, ზრდადობით დალაგებული რიცხვებით გამოსახვა: 'Ex': 5, 'Gd': 4, 'TA': 3, 'Fa': 2, 'Po': 1, 'None': 0

  LotShape-შიც ანალოგიური ლოგიკით: {'Reg': 4, 'IR1': 3, 'IR2': 2, 'IR3': 1}

  ხოლო GarageFinish-ის შემთხვევაში: {'Fin': 3, 'RFn': 2, 'Unf': 1, 'None': 0}

* MoSold, ანუ გაყიდვის თვე, რიცხვითიდან კატეგორიულად ვაქციე, რადგან დეკემბერს (მე-12 თვეს) არ უნდა მიენიჭოს მეტი პრიორიტეტი, ვიდრე იანვარს (1-ელ თვეს). მათი მნიშვნელობები გავხადე სტრინგები: '1', '2', '3' და ა.შ.
  
* სხვა კატეგორიული ცვლადებისთვის გამოვიყენე One-Hot Encoding: 'MSZoning', 'LotConfig', 'Neighborhood', 'Condition1', 'BldgType', 'HouseStyle', 'RoofStyle', 'Exterior1st', 'MasVnrType', 'Foundation', 'BsmtExposure', 'BsmtFinType1', 'BsmtFinType2', 'CentralAir', 'Functional', 'GarageType', 'PavedDrive', 'MoSold', 'SaleType','SaleCondition', ანუ თითო მნიშვნელობისთვის თითო ახალი სვეტი, მაგალითად: MoSold1 (გაიყიდა პირველ თვეში), Mosold2 და ა.შ. ვიფიქრე, რომ გაყიდვის თვე არ იქნებოდა მნიშვნელოვანი ცვლადი, მაგრამ შესაძლოა სეზონური ფასდაკლებები ყოფილიყო, ამიტომ ჯერ თავი შევიკავე დადროპვისგან.

**1. კორელაციების ფილტრი**
დავთვალე ყველა სვეტის კორელაცია target-თან:

დავდროპე ყველა ის ცვლადი, რომელსაც შედეგთან 0,05-ზე ნაკლები კორელაცია აქვს, ანუ საერთოდ არაა თანხვედრაში. მარცხენა გრაფიკზე სრულად არაა მათი სია ნაჩვენები. როგორც მოსალოდნელი იყო, გაყიდვის თვეს საერთოდ არ ჰქონდა გავლენა სახლის ფასთან. 

ასევე დავთვალე კორელაციები ერთმანეთში და იმ წყვილებიდან, რომელთა კორელაციაც 85%-ზე მეტი იყო, დავდროპე რომელიღაც (რომელსაც ნაკლები კორელაცია ჰქონდა თარგეთთან), მაგ: დაიდროპა BsmtFinType2_None და დარჩა BsmtFinType1_None

**2. RFE(Recursive Feature Elimination)**
RandomForestRegressor-ს შევარჩევინე 45 საუკეთესო ცვლადი. RFE-მ ნაბიჯ-ნაბიჯ აკლო სუსტი ცვლადები და ბოლოს გაჩერდა 45-ზე

**3. Lasso Regression**
გამოვიყენე L1 რეგულარიზაცია, რომელმაც ნაკლებგავლენიანი ცვლადების წონები პირდაპირ 0-ზე დაიყვანა და დამიტოვა 34 სვეტი. L1-ის გამოსაყენებლად standard scaling გამოვიყენე, რომ ყველა მნიშვნელობა ერთ მასშტაბში მომექცია.
