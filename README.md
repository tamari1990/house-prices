# house-prices - predicting house prices
## კონკურსის მიმოხილვა
Kaglle house-prices challenge-ის მიზანია სახლის ფასის პროგნოზირება მრავალ ცვლადზე დაყრდნობით: მდებარეობა, ოთახების რაოდენობა, ფართობი, გარაჟი, გაყიდვის ტიპი და ა.შ.

## რეპოზიტორიის სტრუქტურა
```
house-prices/
│
├── model_experiment.ipynb     ← EDA, Lasso Selection, ექსპერიმენტები, XGBoost Tuning
├── model_inference.ipynb      ← საუკეთესო მოდელის ჩატვირთვა რეესტრიდან, პროგნოზი
└── README.md                  ← პროექტის დეტალური აღწერა
```

## ფაილების აღწერა
```
| ფაილი | აღწერა |
| :--- | :--- |
| **model_experiment.ipynb** | მთავარი Notebook მონაცემების დამუშავების, Lasso ფილტრაციისა და სხვადასხვა მოდელების ექსპერიმენტებისთვის. |
| **model_inference.ipynb** | მოდელის Model Registry-დან გამოძახება, ტესტ სეტის მომზადება და Submission-ის გენერაცია. |
```

## მონაცემთა გაწმენდა 
1. **NA-ების მაღალი შემცველობის სვეტების დადროპვა**
   
   იმ სვეტების ახალი მონაცემებით შევსება, რომლებიც თავდაპირველად ძალიან ბევრ გამოტოვებულ მნიშვნელობას შეიცავენ, შეცდომაში შემყვანი შეიძლება აღმოჩნდეს და ისწავლოს     არარეალური კანონზომიერება, რაც overfitting-ის საფრთხეს ზრდის. threshold-ად ავიღე 80%.

   სურათის მიხედვით, PoolQC, MiscFeature, Alley და Fence დაიდროპა.
   წავშალე FireplaceQu-ც, რადგან ძალიან დაწვრილმანებული და უმნიშვნელო ინფორმაცია მგონია.
3. **არაფრისმთქმელი სვეტების დადროპვა**
   
   დავდროპე Id, რადგან იგი უნიკალურია და არავითარი გავლენა არ აქვს სახლის ფასზე
5. **დაბალი განაწილების(ვარიაციის) მქონე სვეტების დადროპვა**
   
   ის სვეტები, რომლებიც დომინირებულია ერთი მნიშვნელობით, უსარგებლოა და არ გვასწავლის კანონზომიერებას.ავაგე გრაფიკი, რომელიც გვაჩვენებს სვეტებს, რომელთა          მნიშვნელობების 90% ერთნაირია.
   

   <img width="1200" height="549" alt="image" src="https://github.com/user-attachments/assets/0d306ce9-d878-4732-9d8d-9862cf86f697" />

   სურათის მიხედვით დავდროპე ისეთი სვეტები, რომელთა გათვალისწინებას დიდად არც ექნებოდა აზრი, ესენია: LowQualFinSF, BsmtHalfBath, MiscVal, KitchenAbvGr, 3SsnPorch, ScreenPorch, Utilities, Street, Condition2, RoofMatl, Heating, PoolArea, LandSlope. რადგანაც, მაგალითად, გათბობა, დენი, სინათლე და სახურავის მასალა თითქმის ყველა სახლს ერთნაირი აქვს, ისინი არ გამოგვადგება ფასის კანონზომიერების დაჭერაში. PoolArea კი იმიტომ დაიდროპა, რომ უმეტესობა სტრიქონი 0-ია, რაც იმაზე მიუთითებს, რომ სახლების უმრავლესობას არც აქვს აუზი(ამაზე მეტყველებდა POOLQC-ში NA-ების სიჭარბეც)

7. **იმ სვეტების დადროპვა, რომელთა სათქმელსაც სხვა სვეტი უკვე ამბობს**
   
   * ყველა სტრიქონში TotalBsmtSF = BsmtFinSF1 + BsmtFinSF2 + BsmtUnfSF. ამიტომ დავდროპე BsmtFinSF1, BsmtFinSF2 და BsmtUnfSF, რადგან ძალიან დაკონკრეტებული ცვლადებია და არაა საჭირო, სარდაფის მთლიანი ფართობიც გვაწვდის საკმარის ინფორმაციას.
   * GarageArea და GarageCars ერთნაირ ინფორმაციას ატარებს, დავდროპე ფართობი და დავტოვე ის, თუ რამდენი მანქანა ეტევა, რადგან ვფიქრობ, უფრო მნიშვნელოვანი და მარტივად აღსაქმელია მყიდველისთვის.
